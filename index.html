Основные концепции MSFvenom

MSFvenom объединяет функциональность двух устаревших инструментов:

  • msfpayload: создание пейлоадов
  • msfencode: кодирование пейлоадов

Ключевые понятия:

  • Payload: код, который выполняется на целевой системе
  • Staged: двухэтапный пейлоад (маленький загрузчик → основной пейлоад)
  • Non-staged: одноэтапный пейлоад (выполняется целиком сразу)
  • Encoder: метод обфускации пейлоада для обхода защитных механизмов

Типы сканирования портов

Nmap поддерживает различные типы сканирования портов:

# TCP SYN сканирование (полуоткрытое, требует прав администратора) nmap -sS 192.168.1.1 # TCP connect сканирование (полное TCP-соединение) nmap -sT 192.168.1.1 # UDP сканирование nmap -sU 192.168.1.1 # Комбинированное TCP и UDP сканирование nmap -sS -sU 192.168.1.1 # Сканирование всех 65535 портов nmap -p- 192.168.1.1 # Сканирование определенных портов nmap -p 22,80,443 192.168.1.1

Какой тип сканирования является наиболее скрытным?

-sT (TCP Connect)
-sS (TCP SYN)
-sU (UDP)
-sA (ACK)

Расширенные возможности Nmap

Определение версий сервисов и операционной системы:

# Определение версий сервисов nmap -sV 192.168.1.1 # Определение ОС nmap -O 192.168.1.1 # Агрессивное сканирование (комбинация опций) nmap -A 192.168.1.1

Скрытое сканирование и уровни времени:

# Скрытое сканирование nmap -T0 192.168.1.1 # Очень медленное nmap -T1 192.168.1.1 # Медленное nmap -T2 192.168.1.1 # Нормальное nmap -T3 192.168.1.1 # По умолчанию nmap -T4 192.168.1.1 # Агрессивное nmap -T5 192.168.1.1 # Самое быстрое

Скрипты NSE и заключение

Nmap Script Engine (NSE) расширяет возможности Nmap:

# Запуск скриптов категории 'vuln' для поиска уязвимостей nmap --script vuln 192.168.1.1 # Запуск конкретного скрипта nmap --script http-title 192.168.1.1 # Запуск нескольких категорий скриптов nmap --script "default,safe,discovery" 192.168.1.1 # Сохранение результатов в файл nmap -A 192.168.1.1 -oN результат.txt

Полезные категории скриптов:

  • default: безопасные скрипты по умолчанию
  • discovery: сбор дополнительной информации
  • safe: скрипты, которые не должны повредить целевой системе
  • vuln: обнаружение уязвимостей
  • exploit: эксплуатация уязвимостей (с осторожностью!)

Процесс установки Kali Linux

Выполните следующие шаги для установки Kali Linux:

  1. Запустите созданную виртуальную машину и выберите загрузку с ISO-образа Kali
  2. Выберите "Graphical Install" в меню загрузки
  3. Следуйте инструкциям установщика: язык, местоположение, раскладка клавиатуры
  4. Настройте учетные данные (имя пользователя и пароль)
  5. При разметке диска выберите "Guided - use entire disk"
  6. Завершите установку и перезагрузите систему

Важно: Используйте сложные пароли. По умолчанию логин kali и пароль kali.

# Обновление системы sudo apt update sudo apt upgrade -y # Установка дополнительных инструментов sudo apt install -y kali-linux-default # Настройка виртуальных дополнений (для VirtualBox) sudo apt install -y virtualbox-guest-x11

  • Изучите базовые команды Linux и файловую систему

Hydra: Тестирование стойкости паролей

Что такое Hydra?

Hydra — это мощный инструмент для тестирования стойкости учетных данных путем проведения брутфорс и словарных атак на сервисы аутентификации.

  • 1
    Поддерживает более 50 протоколов, включая HTTP, FTP, SSH, Telnet, SMB и другие
  • 2
    Позволяет проводить параллельные атаки для ускорения процесса
  • 3
    Включает поддержку прокси и различных форматов словарей

Важно: Использование Hydra без разрешения владельцев систем является незаконным. Применяйте эти навыки только для тестирования собственных систем или с явного разрешения.

Практика: Атака на SSH

Рассмотрим пример атаки на SSH-сервер в тестовой среде:

# Атака по одному пользователю hydra -l root -P /usr/share/wordlists/rockyou.txt 192.168.1.100 ssh # Атака с несколькими пользователями hydra -L users.txt -P passwords.txt 192.168.1.100 ssh # Ограничение попыток (для избежания блокировки) hydra -l admin -P passwords.txt -t 4 192.168.1.100 ssh

Совет: Параметр -t позволяет указать количество параллельных соединений. Использование меньшего значения делает атаку менее заметной, но увеличивает время выполнения.

  • 1

  • 2
    Включает инструменты для всех этапов пентеста: от разведки до постэксплуатации
  • 3
    Предоставляет модульный и расширяемый интерфейс для разработки собственных модулей

Важно: Metasploit — мощный инструмент, который должен использоваться только в легальных сценариях тестирования с явного разрешения владельцев систем.
